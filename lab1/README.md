# Система моделирования киностудии (Лабораторная работа №1)

Данный проект представляет собой программную систему для управления процессами кинопроизводства, разработанную в рамках курса «Проектирование программного обеспечения интеллектуальных систем».

## Предметная область
Система моделирует работу киностудии, включая управление персоналом (режиссеры, актеры), техническими ресурсами (камеры, съемочные площадки) и полным циклом производства фильма: от написания сценария до релиза.

## Основные сущности
В соответствии с требованиями варианта задания реализованы следующие сущности:
* **Киностудия**: Центральный узел управления данными и логикой.
* **Персонал**: Режиссеры (со своими стилями) и Актеры.
* **Ресурсы**: Камеры, Съемочные площадки, отделы Монтажа и Постпродакшна.
* **Производство**: Сценарии и Фильмы с многоступенчатой системой статусов (`ProductionStatus`).

## Реализованные операции
1. **Разработка сценария**: Инициализация нового проекта.
2. **Подбор актеров (Кастинг)**: Проверка совместимости жанра сценария со стилем режиссера и проверка занятости персонала.
3. **Проведение съемок**: Бронирование камеры и площадки на время производства.
4. **Монтаж и Постпродакшн**: Финализация продукта через специализированные отделы.
5. **Релиз фильма**: Завершение проекта и автоматическое освобождение всех ресурсов.

## Технические особенности
* **PEP8 & Типизация**: Весь код написан с соблюдением стандартов оформления и содержит аннотации типов.
* **Обработка исключений**: Реализована иерархия собственных исключений (`StudioError`) для предотвращения логических ошибок.
* **CLI интерфейс**: Взаимодействие с системой осуществляется через текстовое меню в реальном времени.
* **Unit-тестирование**: Проект покрыт тестами с использованием библиотеки `pytest`.

## Структура проекта
Система организована по модульному принципу для обеспечения чистоты кода и разделения ответственности:
* **`studio/people.py`**: Управление кадрами. Реализует базовую логику персон и специфичное поведение режиссеров и актеров с валидацией данных.
* **`studio/resources.py`**: Техническая база. Описывает оборудование (камеры) и локации (площадки), а также логику их занятости.
* **`studio/production.py`**: Процесс производства. Хранит информацию о сценариях, статусах фильма и управляет переходами между этапами продакшна.
* **`studio/exceptions.py`**: Доменные ошибки. Содержит иерархию исключений для обработки некорректных ситуаций в логике студии.
* **`studio/movie_studio.py`**: Главный контроллер. Связывает все компоненты в единый рабочий процесс (Фасад).

## Использование системы (CLI)
После запуска программы (`python main.py`) управление осуществляется через интерактивное меню:
* **Пункты 1-4**: Создание ресурсной базы (наем персонала, покупка техники, постройка площадок).
* **Пункт 5**: Создание сценария (генерация нового кинопроекта).
* **Пункт 6**: Кастинг. Введите ID фильма, ID режиссера и список ID актеров через пробел.
* **Пункт 7**: Съемки. Бронирование камеры и площадки (ресурсы переходят в статус `[BUSY]`).
* **Пункт 8**: Пост-продакшн. Запуск процессов монтажа и визуальных эффектов.
* **Пункт 9**: Релиз. После выхода фильма все связанные ресурсы снова станут доступны (`[FREE]`).
* **Пункт 10**: Просмотр полного списка ресурсов студии и их текущего состояния.
